Slide 1: Introdução ao Novo Modelo de Custos
Objetivo: Expor a fonte de dados e a mudança de paradigma.
Sugestão Visual: Fluxo simples indicando a extração de dados diretamente do Portal Gov.br.
Script de Apoio: O Novo Modelo de Custos proposto fundamenta-se na utilização do Gov.br como fonte primária e primordial de dados, utilizando um dos maiores ativos informacionais que temos na Política Pública de Transformação Digital no Brasil. O diferencial desta abordagem é a eliminação total da dependência de preenchimento manual de formulários pelos gestores dos Órgãos (questão presente nos modelos anteriores). Tudo é feito de forma automática, extraindo os dados diretamente do Portal, criando um mecanismo que evolui sua precisão de forma orgânica: quanto mais os serviços do Gov.br amadurecem e são bem descritos, mais acurados tornam-se os nossos resultados econômicos (oriundos do Novo Modelo de Custos). Importante destacar que há também algumas variáveis globais/complementares que são sazonais e/ou fixas que foram preenchidas e definidas pela própria SGD (como custos globais de TI, média do valor da energia elétrica, média de descolamento urbano e etc.), com base em fontes confiáveis e públicas;
Slide 2: Análise Lexical e as “Macroetapas”
Objetivo: Explicar a lógica de classificação baseada em verbos.
Sugestão Visual: Divisão visual entre as macroetapas "Solicitar" e "Receber" associadas a verbos de ação.
Script de Apoio: Para processar essas informações, utilizamos uma análise lexical das etapas dos serviços digitais descritas no Portal. Entendemos que a descrição dos serviços, especialmente os títulos de suas etapas, de forma geral, possuem uma qualidade de informação aceitável para serem consideradas em uma análise textual. Desta forma, o modelo faz uma varredura no Gov.br e identifica os verbos utilizados no título de cada etapa para cada serviço digital, consolida esses “achados” e então classifica essas etapas em duas “macroetapas” principais e padronizadas que foram definidas pela Novo Modelo de Custos, sendo elas: 'Solicitar' ou 'Receber'. Essa classificação é o que nos permite atribuir valores, calcular tempos médios, consolidar os dados para cada serviço e projetar a visão de economicidade quando comparando aquele serviço se fosse prestado de forma presencial. É uma forma de traduzir a descrição textual do serviço (heterogênea) em uma estrutura de dados organizada e padronizada para o cálculo de custos, sendo, portanto, o serviço (junto de suas etapas), a unidade básica para todos os cálculos.
Slide 3: Variáveis de Mensuração e Volume
Objetivo: Mencionar a existência de múltiplas variáveis sem detalhá-las exaustivamente.
Sugestão Visual: Representação da API de Avaliação e uma menção a variáveis globais e complementares.
Script de Apoio: Além da análise lexical, o modelo incorpora diversas outras variáveis para compor o cálculo. Consideramos aspectos como a geração de documentos, custos de deslocamento e diversas variáveis ambientais e gerais que impactam o custo total. Para parte relevante dessas variáveis, foi utilizado fontes públicas e confiáveis para obtenção dos dados. Uma outra perspectiva importante para essa mensuração é a respeito da quantidade de solicitações dos serviços, sendo essa questão fundamental para os cálculos de atendimentos e impacto econômico dos próprios serviços. Para tanto, utilizamos os dados da nossa API de Avaliação de Serviço para identificar solicitações, tendo em vista que essa API traz quantos solicitações foram geradas para o serviço (solicitações) e quantas converteram em avaliações respondidas. Nos casos em que o serviço ainda não possui essa integração, aplicamos extrapolações estatísticas baseado em médias gerais do Órgão e serviços. Perceba que em alinhamento a premissa de utilização do Gov.br como fonte de dados, quanto mais serviços integrados ao Módulo de Avaliação, melhor também será a acurácia dessa variável de solicitações.
Slide 4: O Serviço como Unidade Básica de Cálculo
Objetivo: Demonstrar o macrofluxo de ponta a ponta usando um exemplo de serviço (visão unitária, o “tijolo”)
Sugestão Visual: O "Serviço" como o bloco fundamental. O macrofluxo: Definição de variáveis globais → Análise do Portal → Catalogação Léxica → Mensuração de Variáveis → Cálculo de Economia.
Script de Apoio: Embora o modelo considere variáveis amplas, a nossa unidade básica de cálculo é o Serviço. Escolhemos essa unidade por ela ser o pilar central de interação no Gov.br e nossa principal fonte de dados. Ao tratarmos o serviço como a célula fundamental, possibilitamos que o modelo seja escalável e replicável para outras perspectivas. Para visualizarmos a aplicação prática, observemos o fluxo de um serviço específico (em linhas gerais):
Iniciamos com as definições de variáveis globais, de ambiente, econômicas e que servirão para sustentar todo o modelo de custos;
Escolhemos um serviço “XYZ” e seguimos para a catalogação léxica (identificação do verbo do título de cada etapa do serviço) de suas etapas no Portal (disponíveis na descrição do serviço). Essa catalogação é feita de forma automática pelo próprio Modelo de Custos, fazendo o uso de um dicionário de dados;
A partir da catalogação, teremos a quantidade de etapas identificadas para cada “macroetapa” do tipo “Solicitar” ou “Receber”. Com a quantidade de etapas do serviço identificadas para cada tipo de “macroetapa”, cruzamos esses dados com as variáveis já definidas e com o volume de solicitações identificadas a partir da API de Avaliação de Serviços;
Para chegarmos então no cálculo de economicidade para o Governo e para a Sociedade, utilizamos os valores obtidos na etapa anterior, junto a alguns outros cálculos complementares e projetamos para um cenário em que aquele mesmo serviço fosse prestado 100% presencial (sem estar digitalizado). A diferença entre os dois cenários representa a economia obtida para aquele serviço.
Esse processo se repete para cada serviço, alimentando nossos painéis estratégicos com dados atualizáveis.
